# HTML学习第二天
今天目标是将HTML学习收尾，进入到css的学习，然后再学习一种设计模式
## 1. 列表
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>列表</title>
</head>
<body>
    <!-- 
        在html中创建列表，分为三种：
            有序列表；
            无序列表；
            定义列表。
        
        有序列表，用ol标签来创建无序列表，li标签表示列表项。
        无序列表，用ui标签来创建无序列表，li标签表示列表项。
        定义列表，用dl标签来创建定义列表，dt标签来表示定义的内容，dd标签表示对内容的解释说明
     
        列表之间可以相互嵌套
    -->
     <ol>
         <li>有序1</li>
         <li>有序2</li>
         <li>有序3</li>
     </ol>

     <ul>
        <li>无序2</li>
        <li>无序2</li>
        <li>无序2</li>
    </ul>

    <dl>
        <dt>定义列表1</dt>
            <dd>dd标签用来对dt解释说明</dd>
            <dd>dd标签用来对dt解释说明</dd>  
        <dt>定义列表2</dt>
            <dd>dd标签用来对dt解释说明</dd>
            <dd>dd标签用来对dt解释说明</dd>  
    </dl>

    <ul>
        <li>1</li>
        <ul>
            <li>2</li>
            <ul>
                <li>3</li>
            </ul>
        </ul>
    </ul>
</body>
</html>
```

## 2. 超链接
```html
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <title>超链接</title>
</head>

<body>
    <!-- 
        使用超链接可以从一个页面跳转到另一个页面，或者是跳转到当前页面的位置
        使用a标签来定义超链接：
            属性：
                href：指定跳转的目标路径
                    可以是一个外部网站的地址，也可以是一个内部页面的地址
                target：指定超链接打开的位置
                    可选择：
                        _self,默认值，在当前页面打开超链接
                        _blank，在一个新页面中打开超链接
    -->
    <a href="https://github.com/Wuczy?tab=repositories" target="_self">橙汁GitHub</a><br>
    <a href="index1列表.html" target="_blank">内部页面</a><br>

    <!--
        可以使用Javascript:;来作为href属性，此时点击这哥超链接什么都不会发生
      -->
      <a href="Javascript:;">超链接</a><br>
    <!-- 
        可以直接将超链接的href属性设置为#，这样点击超链接就会跳转到当前页面的顶部。
        另外，将herf属性设置为目标元素的id属性值，就可以跳转到页面的指定位置
        id属性（唯一不重复的）
            每个标签可以添加一个id属性；
            id属性是元素的唯一标识，同一个页面中不能出现重复的id属性
      -->

    <p>1. 门面模式概述
        1.1 概述
        门面模式又叫作外观模式，提供了一个统一的接口，用来访问子系统中的一群接口。其主要特征是定义了一个高层接口，让子系统更容易使用，属于结构型设计模式。

        其实在日常开发中，我们都有意无意地大量地使用门面模式。但凡只要高层模块需要调度多个子系统（比如2个以上地类对象），我们一般都会地创建一个新类封装这些子系统，提供精简的
        接口，让高层模块可以更加容易地简接调用这些子系统的功能。尤其是现在各种第三方SDK，开源类库，很大部分都会使用门面模式。

        1.2 应用场景
        门面模式不仅在日常开发中常见，在日常生活中门面模式也有很多体现。比如一个网站的首页，一般首页作为整个网站的入口，它提供了很多子栏目的超链接，用户可以通过该首页就可以进
        入各个子栏目的页面，这个首页就好比一个门面；再比如去医院就诊，大部分的医院都设置了导诊台，这样无论我想去皮肤科、内科、外科又或者是妇科，都不需要到处乱转，整个路线变得 非常清楚，这个导诊台也相当于一个门面。

        在日常开发中，门面模式适用于以下场景：

        为一个复杂的模块或子系统提供一个简洁的供外界访问的接口；
        希望提高子系统的独立性；
        当子系统由于不可避免地暂时原因导致可能存在Bug或性能相关问题时，可以通过门面模式提供一个高层接口，隔离客户端与子系统的直接交互，预防代码污染。</p>
```

## 3.图片标签
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>
    <!-- 
        图片标签用于像当前页面中引入一个外部图片
        使用img标签来引入外部图片，img标签是一个自结束标签
        img这种元素属于替代元素，（块元素和行内元素之间，具有两种元素的特点）
        属性：
            src：指定外部图片的路径
            alt：图片的描述，默认不显示，在图片无法加载时会显示此属性值
                搜索引擎会根据alt中的内容来识别图片，如果不设置alt属性则图片不会被搜索引擎所搜索
            width：图片的宽度
            height：图片的高度，单位都是像素，如果宽度和高度都只修改了一个，则另一个等比例修改

            一般情况下，在pc端不建议修改图片的大小，但是在移动端，经常需要对图片进行缩放
                
        图片的格式：
        jpg：支持的颜色比较丰富，不支持透明效果，不支持动图，一般用来显示照片
        gif：支持的颜色比较少，支持简单透明，支持动图，一般用来显示颜色单一，动图
        png：支持的颜色丰富，支持复杂透明，不支持动图，又来显示颜色丰富，复杂透明的照片
        webp：谷歌推出的专门用来表示网页中图片的一种格式，它具备了以上所有优点，并且小
            但是！兼容性不好

        选择的标准：效果一样，用小的；效果不一样，用效果好的。
     -->
    <img src="img/bgimg.jpg" alt="背景图片">
    <img src="img/bgimg1.jpg" alt="背景图片">
    <img src="img/bj.png" alt="">    
</body>
</html>
```

## 4. 内联框架
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>内联框架</title>
</head>
<body>
    <!-- 
        内联框架，用于向当前页面中引入一个其他页面
            src：指定要引入的网页路径
            frameborder：指定内联框架的边框，1表示有边框，0表示无边框
        在实际开发中基本上不使用
     -->
    <iframe src="https://www.bilibili.com/" width="800" height="300" frameborder="1"></iframe>
</body>
</html>
```

## 5. 音视频文件
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>
    <!-- 
        audio标签用来向页面引入一个外部的音频文件
            音视频文件引入时，默认情况下不允许用户自己控制播放停止
        
        属性：
            controls，是否允许用户控制播放，有就允许。
            autoplay，音频文件是否自动播放，如果设置了autoplay则音乐早打开页面时会自动
                    播放，但是目前来讲大部分浏览器不会对音乐自动播放
            loop，音频文件是否循环播放
    -->
    <audio src="source/audio.mp3" controls autoplay></audio>
    <!-- 
        除了通过src来指定外部文件的路径之外，一般通常使用source标签来指定文件路径，解决浏览器的兼容性问题
     -->
     <audio controls>
         <source src="source/audio.mp3">
         <source src="source/audio.ogg">
         <embed src="source/audio.ogg" type="audio/mp3" width="300">
     </audio>
     <!-- 
         使用video标签向网页中引入一个视频
         使用方式与audio方式一样
      -->
      <video controls>
          <source src="source/flower.webm">
          <source src="source/flower.mp4">
          <embed src="source/flower.mp4" type="video/mp4" width="300">
      </video>
</body>
</html>
```
